### **📑 网页设计指令书：直播复盘通用仪表盘 (Live Stream Review Dashboard Template)**

**项目属性：** 动态数据复盘页面
**核心逻辑：** `1个总趋势屏` + `N个单场详情页`（可无限扩展）
**数据更新方式：** 通过配置文件（JSON）或后台录入，前端自动渲染。

---

### **一、 页面整体布局 (Global Layout)**

页面采用 **“上-中-下”** 垂直结构，高度自适应。

#### **1. 顶部：战役总览仪表盘 (Module A: Trend Overview)**

* **功能：** 展示所有场次的宏观走势，一目了然看到业绩曲线。
* **视觉元素：**
* **核心图表：** **折线趋势图 (Line Chart)**
* **X轴 (变量)：** `[直播日期/场次名]` (支持动态增加)
* **Y轴 (变量)：** `[成交单数]` (主坐标轴)，可切换查看 `[场观人数]` 或 `[转化率]`。
* **样式：** 红色平滑曲线，区域填充渐变色（Area Style），高亮显示最高峰值点。


* **关键指标卡 (KPI Cards)：**
* 悬浮在图表上方或右侧，展示：`累计总销售额`、`平均转化率`、`总场观人次`。





#### **2. 中部：场次选择器 (Module B: Session Selector)**

* **功能：** 切换查看不同场次的详细复盘。
* **交互形式：** **时间轴选项卡 (Timeline Tabs)** 或 **横向滚动胶囊按钮**。
* **排序逻辑：** **倒序排列**（最新日期的直播在最左边/最前面，方便查看最新战况）。
* **状态设计：**
* **选中态：** 高亮底色（品牌黄 #FFC000），字体加粗。
* **未选中态：** 灰色底，普通字体。



#### **3. 底部：单场详情复盘容器 (Module C: Detail Container)**

* **功能：** 这是一个**“动态容器”**。根据中部选中的场次，自动渲染该场次的数据和文字。
* **布局结构（左右分栏）：**
* **左侧：数据漏斗区 (Quantitative Data)**
* 设计为**网格布局 (Grid)**，固定显示以下字段（支持空值）：
1. `直播时间` (如：19:00-21:00)
2. `真实场观(UV)` (如：68人)
3. `互动/福袋人数` (如：57人)
4. `成交单数` (如：20单)
5. `转化率` (如：35%)




* **右侧：定性复盘区 (Qualitative Review)**
* 设计为**文本列表布局**，包含三个固定标题的文本块：
1. **✅ 亮点/成功策略** (Highlights)
2. **⚠️ 不足/问题点** (Issues)
3. **💡 迭代/下一步计划** (Action Items)







---

### **二、 数据结构定义 (Data Schema)**

为了方便您后续更新，请告诉开发人员（或您自己编写代码时），数据层必须按照以下**标准结构（JSON Object）**进行设计。

这样您每次直播完，只需要复制一段代码，改改里面的字，图表和详情页就会自动更新。

```json
/* 这是一个标准的数据配置模板，开发者需按照此结构渲染页面 */

var liveSessions = [
  {
    "id": "20260212",             // 唯一ID
    "date": "2月12日",            // 显示在X轴和Tab上的日期
    "title": "年前收官战",        // 场次名称
    "isHighlight": true,          // 是否是高光时刻（UI上加火焰图标）
    
    // --- 顶部图表用的数据 ---
    "chartMetrics": {
      "deals": 20,                // 成交单数 (Y轴数据)
      "viewers": 68               // 场观人数
    },

    // --- 底部详情页用的数据 ---
    "detailData": {
      "time": "18:30-22:00 (3.5h)",
      "realUV": "68人",
      "interaction": "57人 (83%)",
      "conversion": "35%",
      "revenue": "20单 (4999套餐)"
    },

    // --- 具体的文字复盘 ---
    "reviewText": {
      "highlights": [
        "倒计时海报策略成功激活僵尸粉",
        "3.5小时长直播模型验证成功"
      ],
      "issues": [
        "开场前15分钟节奏略拖沓"
      ],
      "actions": [
        "固化'逼单通栏'的设计模板",
        "剪辑高光时刻做短视频分发"
      ]
    }
  },
  
  // 下次直播完，您只需要复制上面这个 { } 块，粘贴在下面，填入新数据即可
  {
    "id": "20260208",
    "date": "2月8日",
    "title": "节前冲刺战",
    // ... 填入2.8的数据 ...
  }
];

```

### **三、 给设计师的视觉规范 (Visual Guidelines)**

1. **扩展性 (Scalability)：**
* 顶部的折线图 X 轴必须支持滚动（万一以后有 20 场直播，不能挤在一起）。
* 中部的 Tab 栏必须支持左右滑动。


2. **空状态 (Empty State)：**
* 如果某场直播的“复盘文字”还没写，不要留白，显示“待复盘”或自动隐藏该板块。


3. **对比度 (Contrast)：**
* **最新一场/选中场次**的数据要用**加粗/深色**显示，其他场次数据稍微变淡，形成视觉层级。
